# LanceDB Web WASM Client

`@lancedb/lancedb-web` is a read-only browser and edge-friendly client for
published Lance tables hosted over HTTP.

It is intentionally narrower than the main JavaScript SDK:

- It opens published table artifacts, not live local or Cloud tables.
- It supports `schema`, `search`, `refresh`, and `close`.
- It executes search in WASM against published metadata and data files.

## Installation

```bash
npm install @lancedb/lancedb-web
```

If you want to generate query embeddings in the client, install the optional
transformers dependency as well:

```bash
npm install @lancedb/lancedb-web @huggingface/transformers
```

## Basic usage

```ts
import { openTable } from "@lancedb/lancedb-web";

const table = await openTable("https://example.com/my_table.lance");

const results = await table.search({
  vector: [0.1, 0.2, 0.3],
  limit: 5,
});

table.close();
```

## Query embedding wrapper

The optional `@lancedb/lancedb-web/transformers` wrapper embeds text queries on
the client and forwards the resulting vector to the base web client.

```ts
import { searchTable } from "@lancedb/lancedb-web/transformers";

const table = await searchTable(
  "https://example.com/my_table.lance",
  "BAAI/bge-small-en-v1.5",
);

const results = await table.search({
  text: "best places to hike in colorado",
  limit: 5,
});
```

## Published table requirements

The browser client expects the published sidecars generated by LanceDB web
publishing:

- `_web.json`
- `_snapshot.json`
- `_latest.manifest`
- `_latest.version`

Published metadata only advertises browser-compatible FTS and vector columns.
Tables that depend on external data or deletion-file bases, or legacy data
files, are rejected by the browser path.

## Runtime notes

- The default path prefers a Web Worker when available.
- The WASM data plane uses `globalThis.fetch` for HTTP range reads.
- In non-browser runtimes, install your fetch implementation on
  `globalThis.fetch` before calling `openTable()`.

## Current limitations

- Read-only only. Writes are out of scope.
- Browser-side vector ranking currently supports `l2`, `cosine`, and `dot`.
- Full-text search is limited to the published FTS columns and scored
  client-side.
- Large tables still depend on HTTP range reads and local ranking work, so this
  path is best suited to published search-oriented snapshots.
